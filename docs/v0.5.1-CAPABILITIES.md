# v0.5.1 — Capability advertisement, machine-readable errors, seeding manifests

Three additive features. All are backwards-compatible and safe to hot-deploy
on any v0.5.0 node.

## 1. Capability advertisement

New endpoints:

- `GET /.well-known/hiverelay.json`
- `GET /api/capabilities` — identical payload, for CDNs/proxies that hide `/.well-known`

**Purpose.** Let clients machine-inspect a relay (version, accept policy, features,
federation state, fees) without opening a Hyperswarm connection. Enables
programmatic relay shopping and monitoring.

**Response shape (schemaVersion 1):**

```json
{
  "schemaVersion": 1,
  "name": "HiveRelay NYC",
  "description": "Public relay for NYC users",
  "icon": "https://example.com/icon.png",
  "pubkey": "<node identity, 32-byte hex>",
  "software": "https://github.com/bigdestiny2/p2p-hiverelay",
  "version": "0.5.1",
  "runtime": "node",
  "region": "us-east-1",
  "contact": "mailto:admin@example.com",
  "terms_of_service": "https://example.com/tos",
  "supported_transports": ["hyperswarm", "dht-relay-ws"],
  "features": [
    "capability-doc",
    "delegation-certs",
    "delegation-revocation",
    "federation"
  ],
  "limitation": {
    "accept_mode": "review",
    "max_pending_requests": 5000,
    "max_connections": 256,
    "max_storage_bytes": 53687091200,
    "max_relay_bandwidth_mbps": 100,
    "delegation_required": false,
    "payment_required": false,
    "auth_required": false
  },
  "federation": { "followed": 12, "mirrored": 2, "republished": 0 },
  "catalog": { "total": 342, "apps": 18, "drives": 280, "resources": 12, "datasets": 20, "media": 12 },
  "fees": null
}
```

**Client usage (new in SDK):**

```js
const caps = await client.fetchCapabilities('https://relay.example.com:9100')
if (caps.limitation.accept_mode === 'open') { /* ...good for publishing */ }
if (caps.features.includes('delegation-revocation')) { /* ...can revoke certs here */ }
```

**Cache.** `Cache-Control: public, max-age=60` — polling every few minutes is fine.

## 2. Machine-readable error prefixes

**Purpose.** Let clients branch on failure type without string-matching human
messages. Each error carries a stable `<kind>: <human message>` prefix.

**Prefixes exported from `p2p-hiverelay/core/error-prefixes.js`:**

| Constant | Prefix |
|----------|--------|
| `ERR.AUTH_REQUIRED` | `auth-required: ` |
| `ERR.PAYMENT_REQUIRED` | `payment-required: ` |
| `ERR.ACCEPT_QUEUED` | `accept-mode-queued: ` |
| `ERR.ACCEPT_REJECTED` | `accept-mode-rejected: ` |
| `ERR.DELEGATION_INVALID` | `delegation-invalid: ` |
| `ERR.DELEGATION_REVOKED` | `delegation-revoked: ` |
| `ERR.DELEGATION_EXPIRED` | `delegation-expired: ` |
| `ERR.RATE_LIMITED` | `rate-limited: ` |
| `ERR.NOT_ALLOWED` | `not-allowed: ` |
| `ERR.NOT_FOUND` | `not-found: ` |
| `ERR.BAD_REQUEST` | `bad-request: ` |
| `ERR.UNSUPPORTED` | `unsupported: ` |

**Client usage:**

```js
import { isErr, classifyErr } from 'p2p-hiverelay/core/error-prefixes.js'

try { await client.publish(...) } catch (err) {
  if (isErr(err, 'AUTH_REQUIRED')) await signIn()
  else if (isErr(err, 'PAYMENT_REQUIRED')) await topUp()
  else if (isErr(err, 'ACCEPT_QUEUED')) showPendingUI()
  else throw err
}
```

**Wire format.** Management endpoints that fail auth return a `errorCode` field
alongside the legacy `error` string:

```json
{
  "error": "Unauthorized — API key required for /api/manage/config",
  "errorCode": "auth-required"
}
```

The `error` string is unchanged — existing clients string-matching on `Unauthorized`
continue to work. New clients should prefer `errorCode`.

## 3. Author seeding manifest

**Purpose.** Authors publish a signed "these are the relays you should fetch my
drives from" document. Complements per-relay `federation.follow()` with a
user-controlled relay-preference primitive.

**New endpoints:**

| Method | Path | Purpose |
|--------|------|---------|
| `POST` | `/api/authors/seeding.json` | Publish a signed manifest (no API key — signature IS authorization) |
| `GET` | `/api/authors/<pubkey>/seeding.json` | Fetch an author's manifest (any cached copy) |

**Client usage:**

```js
// Publish
const manifest = client.createSeedingManifest({
  relays: [
    { url: 'hyperswarm://abc...', role: 'primary' },
    { url: 'wss://relay.example.com/dht', role: 'backup' }
  ],
  drives: [
    { driveKey: '...', channel: 'production' },
    { driveKey: '...', channel: 'beta' }
  ]
})
await client.publishSeedingManifest('https://relay.example.com:9100', manifest)

// Discover
const mine = await client.fetchSeedingManifest('https://relay.example.com:9100', authorPubkeyHex)
if (mine) {
  // Connect to the relays listed in mine.relays to fetch this author's drives
}
```

**Storage.** Relays persist manifests to `storage/manifests.json`. Cap is 10k
authors (oldest-first evicted). Newer-timestamp wins within a given pubkey.

**Signature coverage.** Each manifest covers `type|version|pubkey|timestamp|relays_json|drives_json`
with keys sorted alphabetically, so signature verification is deterministic
across JSON encoders and runtimes.

**Security boundaries.**
- Only the author (holder of the secret key) can sign a manifest under that pubkey.
- Relays cannot tamper with a cached manifest without invalidating the signature.
- Replay protection: manifests with timestamps >5 min in the future are rejected;
  older manifests from the same author replace newer ones only when the new one
  is strictly newer.

## SSH deploy (operators)

All three features are additive — zero config changes required, zero
breaking changes to existing clients.

```bash
# On each relay host:
ssh operator@relay.example.com <<'EOF'
  cd /opt/hiverelay
  git fetch origin
  git checkout release/v0.5.1            # or whatever branch carries this patch
  npm install --omit=dev
  systemctl restart hiverelay
  sleep 2
  # Sanity check:
  curl -s http://127.0.0.1:9100/.well-known/hiverelay.json | jq '{version, features, "accept_mode": .limitation.accept_mode}'
EOF
```

**Rollback.** Checkout the previous tag and restart — the new endpoints are
purely additive; removing them has no effect on any other route.

**Monitoring.** Add a healthcheck that hits `/.well-known/hiverelay.json`. If
it returns 200 with `schemaVersion: 1`, the new surface is live. Existing
monitors on `/health`, `/status`, `/api/overview` are unchanged.
